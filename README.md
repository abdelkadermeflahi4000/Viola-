## ⚡ Quick Start

**Requirements:** Python 3.10+ · Rust + Cargo · pip

```bash
# 1. Clone the repository
git clone https://github.com/your-org/viola
cd viola

# 2. Install Python dependencies
pip install -r requirements.txt

# 3. Build the Rust core (optional — for full performance)
cd viola-rs
cargo build --release
maturin build --release
pip install target/wheels/*.whl
cd ..

# 4. Configure your node
cp viola_config.json.example viola_config.json
# Edit: set your bitcoin RPC credentials and wallet address
nano viola_config.json

# 5. Start resonating
python viola_mining_engine.py
```

Expected output:
```
✨ New node — establishing covenant...
✅ Covenant sealed at Bitcoin block #891,204
🌍 Schumann anchor: 7.83 Hz detected
⬡ Node resonating — TRS: 0.847 [HIGH]
🕊️  Covenant loaded — Node a3f7c9d2...
```

### Run the Simulators

```bash
# Resonance proof-of-concept
python poc_simulator.py

# Schumann integration test
python test_schumann_integration.py

# Full session with bio + cognitive layers
python viola_session.py --mode=resonance

# Hexagonal grid simulation
python unified_hexgrid.py
```

### Docker

```bash
docker-compose up viola-node
```

## 🔬 Open Research Problems

VIOLA is honest about what isn't solved. **These are the open questions that need the community:**

**1. Distributed Clock Synchronization**
Nanosecond-precision agreement between nodes without a central reference. Currently the hardest open problem in the protocol.

**2. Schumann as Timestamp**
The frequency oscillates between 7.2–8.5 Hz due to ionospheric conditions. How do we use it as a reliable anchor despite this natural variance?

**3. Spam Prevention Without Fees**
Without PoW, what's the frequency-based cost model that makes spam economically irrational while keeping the network open?
